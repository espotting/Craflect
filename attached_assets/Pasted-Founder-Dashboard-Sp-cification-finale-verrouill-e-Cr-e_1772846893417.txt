Créer un Founder Dashboard interne permettant de suivre :

* la traction produit
* l’engagement utilisateur
* les revenus
* la progression de l’intelligence du moteur Craflect
* la santé du pipeline

AUTHENTIFICATION

Le Founder Dashboard doit utiliser le système de login existant.

Login unique :

/login

Le login utilisateur reste inchangé.

Après authentification :

* si user.role = admin → accès aux routes admin
* sinon → accès uniquement au produit

Il ne faut PAS créer de login admin séparé.

ROUTES ADMIN

Routes réservées aux admins :

/system/founder
/system/logs
/system/settings

Ces routes doivent être protégées côté backend.

Si user.role ≠ admin :

→ retourner 403 ou rediriger vers /home

VISIBILITÉ INTERFACE

Le lien vers le Founder Dashboard ne doit apparaître dans l’interface que pour les comptes admin.

Dans la sidebar admin :

SYSTEM

* Founder Dashboard
* Logs
* Settings

BACKEND

Créer un endpoint :

GET /api/admin/founder

Accès :

admin uniquement

Cet endpoint retourne toutes les métriques en un seul appel.
